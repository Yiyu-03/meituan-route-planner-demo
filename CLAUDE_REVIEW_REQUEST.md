## 请重点检查的问题

请你不要泛泛夸，也不要建议重做。请检查它是否真的适合给学长/评委第一眼看，并指出“不像美团本地生活产品”“像假 Demo”“体验不顺”的地方。

请重点看这些维度：

1. 第一屏是否能立刻说明：这是一个可直接使用的本地路线方案？
2. 旅行书视觉是否提升了观感，还是只是换皮？
3. 右侧 session 便签是否直观？是否像真实规划历史？
4. 输入区、当前路线、右侧 session 之间是否有状态错位？
5. 页面是否仍然技术感太重？
6. 每站信息是否符合本地生活产品：评分、人均、营业、排队、UGC、订座/导航/收藏？
7. 预算、排队、营业时间等校验文案是否可信？
8. 备选方案是否像真实方案对比，而不是算法输出？
9. 多轮修改入口是否口语化，用户是否知道能怎么改？
10. 评委附录是否既能证明技术链路，又不会干扰普通用户？

## 已知 P0 问题

这些问题大概率需要先修：

1. 预算显示可能误导  
   例如顶部显示 `¥460 ✓`，但预算是 `¥400`，应该显示成 `¥460 / ¥400 · 略超` 或红色提醒，不能打勾。

2. 技术 tag 泄露给用户  
   左下“已识别”里出现 `上海@bund`、`romantic`、`quiet`，应该转为中文自然语言，例如 `外滩`、`情侣氛围`、`安静`。

3. 输入框和当前 session 可能错位  
   输入框可能是“朋友来上海”，但当前右页展示还是“情侣约会”，因为右侧选中了旧 session。需要让用户清楚“当前正在查看哪一次规划”，或者默认输入与当前 session 同步。

4. 预算校验与路线解释有冲突  
   如果路线略超预算，解释里不能说“预算通过”。应该承认略超，并给出替代方案或一键“降预算”。

5. 文案里还有一些不自然的技术味  
   如 `Route Page`、`Itinerary`、`本地生活数据校验`、`评委附录` 等是否需要中文化或弱化，请判断。
